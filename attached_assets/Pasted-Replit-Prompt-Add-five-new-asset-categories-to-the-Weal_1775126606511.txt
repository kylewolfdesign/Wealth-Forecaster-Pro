  ### New interfaces:

  ```ts
  export interface RetirementAccount {
    id: string;
    name: string;
    accountType: '401k' | 'ira' | 'roth_ira' | 'pension' | 'super';
    balance: number;
    monthlyContribution: number;
    employerMatchPct?: number;   // e.g. 50 means employer matches 50% of contributions
    employerMatchCapPct?: number; // max employer match as % of annual salary e.g. 3
    annualSalary?: number;        // required if employerMatchCapPct set
    annualGrowthRate?: number;    // defaults to settings.stockGrowthPct in forecast
  }

  export interface StockOption {
    id: string;
    symbol: string;
    optionType: 'iso' | 'nso';
    totalOptions: number;
    alreadyVestedOptions: number;
    strikePrice: number;
    expirationDate: string; // ISO date string YYYY-MM-DD
    vest: {
      startDate: string;
      cliffMonths: number;
      durationMonths: number;
      frequency: 'monthly' | 'quarterly' | 'yearly';
    };
    growthOverride?: number;
  }

  export interface Bond {
    id: string;
    name: string;
    faceValue: number;
    couponRate: number;       // annual % e.g. 5
    maturityDate: string;     // ISO date string YYYY-MM-DD
    currentValue?: number;    // manual override; falls back to faceValue
    monthlyContribution?: number; // optional reinvestment
  }

  export interface Business {
    id: string;
    name: string;
    currentValue: number;
    ownershipPct?: number;    // display only
    annualGrowthRate?: number; // defaults to 0
    isIlliquid?: boolean;
  }

  export interface Vehicle {
    id: string;
    name: string;
    currentValue: number;
    annualDepreciationRate?: number; // positive number e.g. 15, applied as negative growth.
  Default 15.
  }

  Add helper computeStockOptionVesting (same logic as computeRSUVesting but for StockOption):

  export function computeStockOptionVesting(option: StockOption, atDate: Date): { vested: number;

  Use the identical vesting algorithm as computeRSUVesting, substituting option.totalOptions,

  Update computeCurrentTotals signature:

  export function computeCurrentTotals(
    holdings: Holding[],
    rsuGrants: RSUGrant[],
    cashAccounts: CashAccount[],
    mortgages: Mortgage[],
    otherAssets: OtherAsset[],
    realEstate: RealEstate[],

    stockOptions: StockOption[],
    bonds: Bond[],
    businesses: Business[],
    vehicles: Vehicle[],
  ): SnapshotTotals

  Add to computeCurrentTotals body:
  - retirement: sum of account.balance for all retirementAccounts
  - stockOptions: sum of max(0, (getInstantPrice(opt.symbol, 'stock') - opt.strikePrice) *
  computeStockOptionVesting(opt, now).vested) for all stockOptions. Return 0 if expirationDate has
  passed.
  - bonds: sum of (bond.currentValue ?? bond.faceValue) for all bonds
  - business: sum of b.currentValue for all businesses
  - vehicles: sum of v.currentValue for all vehicles

  Include all five in totalAssets calculation. Add all five fields to the returned SnapshotTotals
  object.

  Update computeForecast signature to accept the five new arrays (same pattern as existing args).
  Add forecast logic for each:

  - retirement: use growWithContributions. Monthly contribution = monthlyContribution +
  monthlyEmployerMatch. Monthly employer match = min(monthlyContribution * (employerMatchPct/100 ??
   0), (annualSalary ?? 0) * (employerMatchCapPct/100 ?? 0) / 12). Growth rate = annualGrowthRate
  ?? settings.stockGrowthPct.
  - stockOptions: at each forecast point, compute vested options using
  computeStockOptionVesting(opt, futureDate). Project price with growMonthly. Value = max(0,
  projectedPrice - strikePrice) * vested. Zero out options past expirationDate.
  - bonds: use growWithContributions(bond.currentValue ?? bond.faceValue, bond.monthlyContribution
  ?? 0, bond.couponRate, months). At or after maturityDate, lock value to bond.faceValue.
  - business: growMonthly(b.currentValue, b.annualGrowthRate ?? 0, months)
  - vehicles: Math.max(0, growMonthly(v.currentValue, -(v.annualDepreciationRate ?? 15), months))

  Add all five to the ForecastPoint breakdown and to totalAssets. Update the existing initial point
   (monthsFromNow: 0) to include the five new breakdown fields.
